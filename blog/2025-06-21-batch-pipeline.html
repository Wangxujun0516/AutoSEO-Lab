批量流水线与结构化导出：工业级 CSV 审计报告 ​

单件清洗只是内功，真正的生产力来自批量编排。 本文展示如何把 Shadow Ledger 提取器套进循环，加上 CSV I/O、计数器和状态报告，打造一条微型但完整的工业级流水线。

从单件到批量：架构设计 ​

一个完整的批量流水线需要五个组件：

Input (CSV) → Extract (Shadow Ledger) → Clean (Pipeline) 
                                              ↓
Output (CSV) ← Export (Structured) ← Audit (Statistics)
组件职责文件
Input读取原始 URL 与 HTMLinput.csv
Extract从 HTML 中提取标题fetch_title_pro.py
Clean应用配置化的清洗规则batch_title_cleaner.py
Audit统计变更、标记失败内嵌于 batch 函数
Export写入结构化 CSV 报告seo_metadata_exporter.py

批量处理核心实现 ​

python
import csv
from pathlib import Path


def batch_clean(input_path: str, output_path: str, **kwargs) -> list[dict]:
    """
    Read CSV, clean titles, write output CSV.
    
    Args:
        input_path: Path to input CSV with columns [url, html]
        output_path: Path to write output CSV
        **kwargs: Cleaning configuration forwarded to clean_title()
    
    Returns:
        List of processed row dictionaries
    """
    rows = []
    
    # ── Phase 1: Read ──
    with open(input_path, "r", encoding="utf-8") as f:
        reader = csv.DictReader(f)
        for row in reader:
            original = row.get("title", "") or extract_title_from_html(row.get("html", ""))
            cleaned = clean_title(original, **kwargs)
            rows.append({
                "url": row.get("url", ""),
                "original_title": original,
                "cleaned_title": cleaned,
            })
    
    # ── Phase 2: Write ──
    Path(output_path).parent.mkdir(parents=True, exist_ok=True)
    with open(output_path, "w", newline="", encoding="utf-8") as f:
        w = csv.DictWriter(f, fieldnames=["url", "original_title", "cleaned_title"])
        w.writeheader()
        w.writerows(rows)
    
    # ── Phase 3: Audit ──
    changed = sum(1 for r in rows if r["original_title"] != r["cleaned_title"])
    print(f"[+] Cleaned {len(rows)} titles ({changed} modified) -> {output_path}")
    
    return rows

关键设计决策 ​

newline=""：这是工业级 CSV 写入的标准做法。在 Windows 上，newline="" 防止 csv.writer 把 \n 转换成 \r\n\r\n，避免产生空行。

encoding="utf-8"：强制使用 UTF-8，支持中文、日文、emoji 等所有 Unicode 字符。BOM-free，兼容 Linux/macOS/Windows 所有平台。

Path.parent.mkdir(parents=True)：自动创建输出目录，无需手动确保路径存在。

工业级写入实现 ​

python
import csv
from datetime import datetime


def export_to_seo_sheet(dataset: list[dict], filename: str) -> dict:
    """
    Write sanitized titles into a structured CSV audit report.
    
    Args:
        dataset: List of {url, html} dictionaries
        filename: Output CSV path
    
    Returns:
        Statistics dictionary with total/success/failed counts
    """
    fieldnames = ["Task_ID", "Status", "Extracted_Meta_Title", "URL", "Timestamp"]
    stats = {"total": 0, "success": 0, "failed": 0}
    
    # newline="" is the industrial standard for safe CSV writing on all OSes
    with open(filename, mode="w", newline="", encoding="utf-8") as csv_file:
        writer = csv.DictWriter(csv_file, fieldnames=fieldnames)
        writer.writeheader()
        
        for current_id, item in enumerate(dataset, start=1):
            title = get_clean_title(item.get("html", ""))
            status = "SUCCESS" if "错误" not in title else "FAILED"
            
            writer.writerow({
                "Task_ID": f"SEO-TASK-{current_id:03d}",
                "Status": status,
                "Extracted_Meta_Title": title,
                "URL": item.get("url", ""),
                "Timestamp": datetime.now().isoformat()
            })
            
            stats["total"] += 1
            stats["success" if status == "SUCCESS" else "failed"] += 1
    
    # Print audit summary
    success_rate = stats["success"] / stats["total"] * 100 if stats["total"] else 0
    print(f"\n{'='*50}")
    print(f"SEO Audit Report: {filename}")
    print(f"{'='*50}")
    print(f"Total:    {stats['total']:>4}")
    print(f"Success:  {stats['success']:>4} ({success_rate:.1f}%)")
    print(f"Failed:   {stats['failed']:>4}")
    print(f"{'='*50}\n")
    
    return stats

命令行接口：从脚本到工具 ​

一个工业级的工具必须有友好的 CLI。使用 Python 的 argparse 模块：

python
import argparse
import sys


def main():
    parser = argparse.ArgumentParser(
        description="Batch title cleaner for SEO audit"
    )
    parser.add_argument("--input", "-i", required=True, help="Input CSV path")
    parser.add_argument("--output", "-o", required=True, help="Output CSV path")
    parser.add_argument("--strip-suffix", help="Suffix to remove (e.g., ' | Brand')")
    parser.add_argument("--strip-emoji", action="store_true", help="Remove emoji")
    parser.add_argument("--case", choices=["lower", "upper", "title", "sentence"],
                        help="Convert case")
    parser.add_argument("--max-len", type=int, help="Max title length")
    parser.add_argument("--audit", action="store_true", help="Generate audit report")
    
    args = parser.parse_args()
    
    kwargs = {
        "strip_suffix": args.strip_suffix,
        "strip_emoji": args.strip_emoji,
        "case": args.case,
        "max_len": args.max_len,
        "collapse_whitespace": True
    }
    
    # Remove None values
    kwargs = {k: v for k, v in kwargs.items() if v is not None}
    
    if args.audit:
        # Read input for audit mode
        dataset = []
        with open(args.input, "r", encoding="utf-8") as f:
            reader = csv.DictReader(f)
            for row in reader:
                dataset.append({
                    "url": row.get("url", ""),
                    "html": row.get("html", "")
                })
        export_to_seo_sheet(dataset, args.output)
    else:
        batch_clean(args.input, args.output, **kwargs)


if __name__ == "__main__":
    main()

流水线扩展：从清洗到分发 ​

当前的流水线覆盖了 Phase 1（数据清洗）。向 Phase 2 和 Phase 3 扩展时，可以复用相同的架构：

Phase 2: 数据采集
  └─ GSC Orchestrator → Proxy Handler → Rate Limiter → Raw JSON

Phase 3: 索引分发
  └─ Sitemap Parser → URL Batch → Indexing API / IndexNow → Status Log

每个阶段都输出标准化的 CSV/JSON，供下一阶段消费。这种"管道-过滤器"架构确保了模块间的松耦合。

写在最后 ​

批量流水线是"自动化"的真正起点。单件函数的优雅只是基础，把它们串联成可配置、可观测、可审计的完整系统，才是工程化的标志。

本文介绍的流水线虽然微型，但已经具备了工业级工具的核心特征：

  • 确定性：相同的输入总是产生相同的输出
  • 可复现：CSV 中间件让每一步都可回溯
  • 可审计：统计报告和状态标记让问题一目了然

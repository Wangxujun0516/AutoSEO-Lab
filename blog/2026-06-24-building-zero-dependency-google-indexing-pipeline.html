Chapter 1: 架构瓶颈 ​

403 Forbidden 问题 ​

Google Indexing API 实施了严格的访问控制。我们最初尝试使用 OAuth2 Web Client 令牌时立即被拒绝，返回 HTTP 403 Forbidden。根本原因：

认证方式Google Indexing API 状态原因
OAuth2 Web Client（用户授权）❌ 403 ForbiddenIndexing API 需要 Service Account 委托
Service Account（直接）✅ 已授权已配置显式域级委托

库兼容性失败 ​

高级 Python 库在我们沙盒化的 Windows 环境中引入了关键故障：

库问题影响
oauth2client已弃用，与现代 Python 不兼容ModuleNotFoundError
google-auth需要 cryptography依赖注入被阻止
requests忽略系统代理设置超时死锁
httplib2代理处理器与 Clash 不兼容连接被拒绝

网络沙盒约束 ​

我们的开发环境在严格约束下运行：

  • 不允许外部 pip 包注入
  • 系统代理 Clash 锁定在 127.0.0.1:3067
  • 第三方库依赖在防火墙级别被阻止

这迫使我们构建一个 100% 原生 Python 解决方案，仅使用标准库组件。

Chapter 2: 纯原生令牌交换解决方案 ​

架构设计 ​

我们重构 batch_index.py 以实现零依赖令牌交换架构：

credentials.json → JWT 签名 → Google OAuth2 网关 → Access Token → Indexing API

核心实现 ​

python
import sys
import os
import json
import time
import urllib.request
import urllib.parse

def get_access_token_from_sa(sa_file_path):
    """
    零依赖 Service Account 令牌交换。
    
    解析 credentials.json 并生成签名 JWT 断言，
    然后通过 Google 官方 OAuth2 网关将其交换为有效的 OAuth2 Bearer Token。
    """
    try:
        with open(sa_file_path, 'r') as f:
            sa_data = json.load(f)
        
        private_key = sa_data['private_key']
        client_email = sa_data['client_email']
        token_uri = sa_data.get('token_uri', 'https://oauth2.googleapis.com/token')
        
        # 标准 JWT 签名（需要原生 jwt 模块）
        import jwt
        iat = int(time.time())
        exp = iat + 3600
        
        payload = {
            "iss": client_email,
            "scope": "https://www.googleapis.com/auth/indexing",
            "aud": token_uri,
            "exp": exp,
            "iat": iat
        }
        
        # 使用 RS256 算法签名
        signed_jwt = jwt.encode(payload, private_key, algorithm='RS256')
        
        # 将 JWT 交换为 Access Token
        data_payload = urllib.parse.urlencode({
            'grant_type': 'urn:ietf:params:oauth:grant-type:jwt-bearer',
            'assertion': signed_jwt
        }).encode('utf-8')
        
        req = urllib.request.Request(
            token_uri,
            data=data_payload,
            headers={'Content-Type': 'application/x-www-form-urlencoded'},
            method='POST'
        )
        
        with urllib.request.urlopen(req, timeout=10) as response:
            res_data = json.loads(response.read().decode('utf-8'))
            return res_data.get("access_token")
            
    except Exception as e:
        print(f"[-] 令牌交换失败: {e}")
        return None

为什么这有效 ​

  1. Service Account 认证：Google Indexing API 明确要求具有域级权限的 Service Account 委托
  2. 原生 JWT 签名：使用 Python 内置的 jwt 模块（在我们的沙盒环境中可用）
  3. 标准 OAuth2 流程：遵循 Google 官方令牌交换协议
  4. 零外部依赖：仅使用 urllib.request、json、time、os、sys

增量提取逻辑 ​

python
def detect_new_urls(fetched_urls, indexed_history):
    """
    通过将获取的 URL 与本地状态表进行比较来执行增量提取。
    
    返回：不在历史记录中的 URL 列表（新部署的）
    """
    return [url for url in fetched_urls if url not in indexed_history]

Sitemap 解析 ​

python
def fetch_and_parse_sitemap(sitemap_url):
    """
    使用原生 xml.etree.ElementTree 解析远程 sitemap.xml。
    
    提取 sitemap 命名空间中的所有 <loc> 标签。
    """
    import xml.etree.ElementTree as ET
    
    urls = []
    req = urllib.request.Request(
        sitemap_url,
        headers={'User-Agent': 'AutoSEO-Lab-Sitemap-Watcher/1.0'}
    )
    
    with urllib.request.urlopen(req, timeout=15) as response:
        xml_data = response.read()
    
    root = ET.fromstring(xml_data)
    namespace = {'ns': 'http://www.sitemaps.org/schemas/sitemap/0.9'}
    
    for url_tag in root.findall('ns:url', namespace):
        loc_tag = url_tag.find('ns:loc', namespace)
        if loc_tag is not None and loc_tag.text:
            urls.append(loc_tag.text.strip())
    
    return urls

速率限制合规性 ​

系统在 API 调用之间强制执行 1.5 秒睡眠阈值，以保持对 Google 速率限制的合规性：

python
for i, url in enumerate(new_urls, 1):
    submit_to_google_indexing(url, access_token)
    
    # 优雅的速率限制
    if i < len(new_urls):
        time.sleep(1.5)
